1 Vue d'ensemble

Slapppy est une app macOS de barre de menus qui permet d'assigner des actions (raccourcis clavier, texte) à des patterns de slaps (séquences rythmiques sur le trackpad) et à des gestes tracés (formes dessinées avec ⌥ + drag).

Cas d'usage typique
L'utilisateur tape trois fois rapidement sur son MacBook → Slapppy reconnaît le pattern → exécute F13 → déclenche un raccourci Keyboard Maestro qui ouvre Zoom.

Technologies clés

TechnologieUsage
IOKit / HIDAccès direct à l'accéléromètre Bosch BMI286 (Apple Silicon)
NSEvent global monitorCapture des taps ⌥ et mouvements trackpad
CGEventSimulation de frappes clavier au niveau système
SwiftUI + ObservableUI menu bar réactive
Sparkle 2.9Mises à jour automatiques hors App Store
$1 Unistroke RecognizerReconnaissance de gestes invariante en rotation/échelle

Permissions requises

  • Input Monitoring — écouter les événements clavier/souris globaux
  • Accessibilité — poster des CGEvent au niveau système

L'app est non-sandboxée (obligatoire pour IOKit et CGEvent).

3 Architecture

L'app suit une architecture service + injection d'environnement. Tous les services sont instanciés dans SlappyApp comme @State et injectés via .environment().

// SlappyApp.swift — injection de tous les services
ContentView()
    .environment(updater)       // Sparkle
    .environment(trackpad)      // Événements trackpad
    .environment(recorder)      // Enregistrement patterns
    .environment(store)         // Persistance patterns
    .environment(engine)        // Matching patterns
    .environment(gestureRecorder)
    .environment(gestureStore)
    .environment(gestureEngine)

Câblage des callbacks

TrackpadReader.onTap(intensity) ──► PatternRecorder.handleSlap()
                               ──► PatternEngine.handleSlap()

TrackpadReader.onGesture(pts) ──► GestureRecorder.handleGesture() (si recording)
                              ──► GestureEngine.handleGesture() (sinon)

PatternEngine.onMatch(pattern) ──► ActionExecutor.execute(action)
GestureEngine.onMatch(template) ──► ActionExecutor.execute(action)

AccelerometerReader.swift
Accès direct IOKit à l'accéléromètre Bosch BMI286 (Apple Silicon)

Lit les rapports HID bruts à 1 kHz. Détecte les slaps par envelope-tracking :

Signal IMU (1 kHz) ──► EMA lente (baseline gravité)
                  ──► Déviation positive = signal - EMA
                  ──► Noise envelope (bruit ventilateur/bobines)
                  ──► Slap si déviation > envelope + 100 (+ 0.3s réfractaire)
ConstanteValeurRôle
kNoiseGate150.0Seuil sous lequel une déviation est classée bruit
kNoiseEnvelopeDecay0.999Décroissance enveloppe (~700ms demi-vie)
kSlapMargin100.0Marge au-dessus de l'enveloppe pour déclencher
kSlapRefractory0.3sDélai minimum entre deux slaps
kEMAAlpha0.001Constante de temps EMA baseline (~1s)
kEMAWarmup500 samplesPhase de chauffe (~0.5s) avant la détection
👆
TrackpadReader.swift
Monitor global NSEvent — taps ⌥ et tracés de gestes

Enregistre des monitors globaux et locaux pour [leftMouseDown, leftMouseUp, mouseMoved, flagsChanged].

Intensité des taps

Dérivée de la durée contact (mouseDown → mouseUp) :

let intensity = max(400, min(1200, 400 + ms × 5.3))
// tap léger ~20-50ms  → intensity ~400
// clic fort ~150-200ms → intensity ~1200

Capture de gestes

Accumulation de deltaX / deltaY relatifs (indépendant de la vitesse curseur système) :

// ⌥ pressé → initialisation à (0,0)
gesturePoints = [.zero]

// mouseMoved → accumulation des deltas
gesturePoints.append(CGPoint(
    x: last.x + event.deltaX,
    y: last.y - event.deltaY  // deltaY positif = vers le bas (AppKit)
))

🔬
DollarOneRecognizer.swift
Implémentation Swift du $1 Unistroke Recognizer (Wobbrock et al., UIST 2007)

Pipeline de normalisation (reconnaissance)

  1. Rééchantillonnage à 64 points équidistants
  2. Rotation pour aligner centroïde → premier point à l'angle 0
  3. Scale non-uniforme vers une boîte 250×250
  4. Translation du centroïde à l'origine

Pipeline d'affichage (display)

  1. Rééchantillonnage à 64 points
  2. Scale uniforme (ratio préservé)
  3. Translation à l'origine — sans rotation, sans déformation

Matching — golden-section search

Recherche de la meilleure rotation dans ±45° qui minimise la distance moyenne point-à-point entre le candidat et chaque template. Score = 1 − bestDist / halfDiagonal ∈ [0,1]. Seuil de match : 80%.

7 Décisions d'architecture clés

Choix techniques structurants du projet, avec leur justification.

1. Envelope-tracking pour la détection de slaps

Le bruit de fond (ventilateur, bobines magnétiques) crée une oscillation permanente de l'accéléromètre. Une simple valeur seuil fixe aurait généré des faux positifs. L'envelope-tracking adapte le seuil dynamiquement à l'environnement courant — fonctionnel aussi bien dans le silence qu'avec un Mac sous charge.

2. Durée de contact comme proxy d'intensité

Force Touch n'est pas disponible sur tous les MacBook et n'est pas accessible par API publique. La durée mouseDown→mouseUp (20–200ms) donne une variation d'intensité fiable sur tout MacBook, sans dépendance hardware spécifique.

3. Deux chemins de normalisation pour les gestes

Le $1 Recognizer normalise en rotation + scale non-uniforme pour maximiser la robustesse de la reconnaissance. Mais ce même traitement déforme visuellement les tracés (un cercle devient carré). Deux versions sont donc stockées : path pour la reconnaissance, displayPath pour l'affichage fidèle.

4. Deltas relatifs trackpad plutôt que position curseur

NSEvent.mouseLocation dépend de la vitesse du pointeur (préférences système) et de l'accélération. Les deltaX/deltaY capturent le mouvement réel du doigt, ce qui rend les gestes cohérents quelle que soit la configuration système.

5. Non-sandboxé par nécessité

Trois fonctionnalités requièrent de sortir du sandbox : IOKit HID (accéléromètre), CGEvent au niveau système (simulation clavier), et NSEvent global monitor (écoute d'événements hors app). Distribution hors App Store (Developer ID) uniquement.

6. Fenêtre glissante avec éviction temporelle

Les slaps sont maintenus dans une fenêtre de 10 événements max / 10 secondes. Les slaps "anciens" sont évictés automatiquement pour ne pas contaminer la reconnaissance d'une nouvelle séquence démarrée après une pause.

7. Cooldown par pattern/geste

1.5 secondes de réfractaire par template évite les doubles déclenchements quand la fenêtre glissante contient plusieurs sous-séquences qui matchent le même pattern consécutivement.

8. Sparkle hors App Store

Sans App Store, les mises à jour automatiques nécessitent un framework tiers. Sparkle est le standard de facto pour les apps macOS indépendantes. La signature EdDSA garantit l'intégrité des DMG distribués via GitHub Releases.
